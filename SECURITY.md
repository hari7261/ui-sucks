# Security Policy

## Supported Versions

Security fixes are applied to the latest published version on npm and the `main` branch on GitHub.

## Reporting a Vulnerability

Do not open a public GitHub issue for security-sensitive reports.

Instead:

1. Email the maintainer at `hariompandit5556@gmail.com`
2. Use the subject line `ui-sucks security report`
3. Include reproduction steps, impact, and any proof-of-concept details

You can expect an initial acknowledgement within 72 hours.

## Disclosure Process

- Reports are validated privately
- A fix is prepared and tested
- A patched release is published
- Public disclosure follows once users have a remediation path
